Resumen técnico y arquitectura

MercApp está dividida en dos capas desplegadas de forma independiente. El frontend es una SPA en Vue 3 que consume el backend mediante una URL pública configurable. El backend corre con Express, usa MongoDB Atlas como fuente de verdad y expone un API REST para productos y categorías.

  • Frontend: Vue 3 + Vue Router + Vite, desplegado en Netlify.
  • Backend: Node.js + Express + MongoDB, desplegado en Railway.
  • Persistencia: colecciones `categories` y `products` en MongoDB Atlas.
  • Consumo de API: el frontend centraliza las llamadas en una única capa de servicio.

Guía breve de ejecución local

Para levantar el proyecto localmente necesitas dos terminales: una para el backend y otra para el frontend. El frontend usa la variable `VITE_API_URL` para apuntar al API publicado.

Backend
cd backend
npm install
npm start

Frontend
cd frontend
npm install
npm run dev
Nota: si cambias la URL del API, actualiza `frontend/.env.production` y vuelve a desplegar el frontend.

Endpoints principales

Tabla corta con método, ruta y un ejemplo de petición/respuesta.

Método Ruta Ejemplo Respuesta
GET /api/health fetch('/api/health') { ok: true, message: 'MercApp API funcionando' }
GET /api/categories fetch('/api/categories') [{ id, name }]
GET /api/products fetch('/api/products') [{ id, name, price, stock, ... }]
GET /api/products/:id fetch('/api/products/2') { id: 2, name: 'Audifonos Pulse', ... }
POST /api/products { "name": "Nuevo", "price": 10, ... } { id: 11, name: 'Nuevo', ... }
PUT /api/products/:id PUT /api/products/2 { id: 2, name: 'Actualizado', ... }
PATCH /api/products/:id { "stock": 12 } { id: 2, stock: 12, ... }
DELETE /api/products/:id DELETE /api/products/2 { message: 'Producto eliminado correctamente.' }

Activar GitHub Pages y validar HTTPS

  1. En GitHub, entra al repositorio y abre Settings.
  2. Ve a Pages y elige Deploy from a branch.
  3. Selecciona la rama principal y la carpeta /docs.
  4. Guarda los cambios y espera a que GitHub genere la URL pública.
  5. Verifica que la página cargue por HTTPS y que el enlace sea accesible sin autenticación.
